dapp

Build, test, deploy and transact with Ethereum contracts from the comfort of the command-line.

Quick Usage

$ dapp init                  create a new dapp

$ dapp build                 compile the source code
$ dapp test                  run the test suite

$ dapp install <pkg>         install a dappsys package
$ dapp upgrade <pkg>         upgrade a package
$ dapp uninstall <pkg>       uninstall a package

$ dapp create <args>         deploy a contract

Quick Install

We recommend using the Nix package manager. It works on Linux and OS X and is completely unobtrusive. We maintain our own channel for distributions of our software along with the Solidity compiler (multiple versions). 

# Install Nix:
curl https://nixos.org/nix/install | sh

# Add the DappHub channel to Nix:
nix-channel --add https://nix.dapphub.com/pkgs/dapphub
nix-channel --update

# Install one or many tools as required:
nix-env -iA dapphub.{dapp,hevm,seth,solc}
See the full install instructions for more detailed guide.

DappHub

DappHub is a community of developers, computer scientists, security researchers, type theorists working on blockchain software. Our suite of tools has grown through our work on Maker and other consulting projects and designed to ease the correct construction and deployment of distributed applications.

Our open source toolkit includes:

  • dapp build, test, deploy and transact with Ethereum contracts from the comfort of the command line.
  • seth, a utility knife for slicing and dicing transactions, performing remote calls, and converting between data formats; and
  • hevm, an EVM implementation designed for testing and verification, with a nimble debugger that runs in the terminal.

We like tools that are fast, transparent, and good citizens of the command line, which is how we've built the dapp suite.

DappSys - Smart contract building blocks

We also maintain Dappsys - a pattern language of contracts with formally verified semantics. It includes
  • ds-token, a generic EIP-20 coin;
  • ds-cabal, a multisig;
  • ds-guard, a flexible authority rule;
  • ds-proxy, a transaction proxy; and
  • ds-cache, a store of expiring values.

Using these proven parts lets us focus on the novel features of the systems we develop. We share Dappsys under a free software license to benefit the smart contract ecosystem.
