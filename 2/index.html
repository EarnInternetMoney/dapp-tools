Tools for dapps

Developing for Ethereum demands a suite of tools to help with construction, correctness, deployment, and management.

The open source toolkit by DappHub consists of

  • dapp, a command line tool that initializes, compiles, links, tests, deploys, and interacts with Ethereum contracts;
  • seth, a utility knife for slicing and dicing transactions, performing remote calls, and converting between data formats; and
  • hevm, an EVM implementation designed for testing and verification, with a nimble debugger that runs in the terminal.

We prefer tools that are fast, transparent, and good citizens of the command line, and that's how we've built the dapp suite.

These tools grew out of our work on Maker and other consulting projects.

Verified contract components

The other half of our development strategy is Dappsys, a pattern language of contracts with formally verified semantics. It includes
  • ds-token, a generic EIP-20 coin;
  • ds-cabal, a multisig;
  • ds-guard, a flexible authority rule;
  • ds-proxy, a transaction proxy; and
  • ds-cache, a store of expiring values.

Using these proven parts lets us focus on the novel features of the systems we develop.

We share Dappsys under a free software license to benefit the smart contract ecosystem.

Installing the DappHub toolkit

We recommend using our installer. The system is nonintrusive and can be completely uninstalled later. 

$ curl https://dapp.tools/install | sh
(1/4) Downloading...
(2/4) Installing Nix environment...
(3/4) Refreshing package channel...
(4/4) Installing base packages...

Done! Try `dapp new my-dapp' or `dapp help'.
If you'd rather install manually, the script serves as documentation.

You can switch to the latest release of the toolkit: 

$ dapp self-upgrade
Installed new versions:
  dapp v1.2    (was v1.1)
  solc v0.4.6  (was v0.4.5)

You can also uninstall it. After this there should be no trace of the system left on your computer. 

$ dapp self-destroy
Dapp tools removed.

Introduction to the Nix package manager

The dapp tool builds on Nix, a package manager with properties that make it uniquely robust, far beyond tools like npm, and uniquely self-contained.

You don't need knowledge of Nix to operate dapp, but we recommend anyone interested in system administration or software deployment to learn about it.

Nix's robustness is due to its use of cryptographic digests to track dependencies and configurations. This fits naturally with the blockchain paradigm—we're in the age of hashes.

Eelco Dolstra's 2006 Ph.D. thesis, "The Purely Functional Software Deployment Model", describes the system in detail. Print it out for a rainy day—it's worth reading.
